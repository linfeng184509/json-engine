## 足不出户

能在这个页面解决的问题，就不要去其它页面解决。

### 问题背景

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   页面刷新和跳转 → 变化盲视 → 心流被打断                  │
│                                                         │
│   变化盲视 (Change Blindness):                          │
│   视觉场景中的某些变化并未被观察者注意到                   │
│                                                         │
│   心流 (Flow):                                          │
│   将个人精神力完全投注在某种活动上的感觉                   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 解决方案

#### 1. 覆盖层 (Overlay)

**二次确认覆盖层**

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   推荐：直接操作 + 撤销                                  │
│                                                         │
│   ┌─────────────────────────────────────┐               │
│   │  删除成功 [撤销]                    │  ← Message    │
│   └─────────────────────────────────────┘               │
│                                                         │
│   而非：滥用 Modal 确认                                  │
│                                                         │
│   特例：无法撤销的操作使用 Popconfirm                     │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**详情覆盖层**

| 触发方式 | 场景 |
|---------|------|
| 悬停 | 预览信息，快速查看 |
| 点击 | 需要操作的详情 |

**注意事项**：
- 悬停触发需 0.5s 延迟
- 移出时立刻关闭

**输入覆盖层**

- 覆盖层上直接输入
- 点击外部自动保存退出

#### 2. 嵌入层 (Embedded Layer)

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   列表嵌入层：                                           │
│   ┌─────────────────────────────────────┐               │
│   │  列表项 1                           │               │
│   ├─────────────────────────────────────┤               │
│   │  列表项 2                           │               │
│   │  ┌───────────────────────────────┐  │  ← 展开的详情  │
│   │  │        详情内容               │  │               │
│   │  └───────────────────────────────┘  │               │
│   ├─────────────────────────────────────┤               │
│   │  列表项 3                           │               │
│   └─────────────────────────────────────┘               │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**标签页**：将平级信息分类，一次显示一组

#### 3. 流程处理

**渐进式展现**

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   基于用户操作，渐进展现不同选项                          │
│                                                         │
│   状态 1: [基础选项]                                     │
│              ↓ 选择高级                                  │
│   状态 2: [基础选项] + [高级选项]                        │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**配置程序**：通过配置项帮助用户完成任务

**步骤条**：复杂流程在 Modal 中使用步骤条管理

## 简化交互

根据费茨法则，用户鼠标移动距离越短、对象相对目标越大，越容易操作。

### 费茨法则

```
公式: T = a + b × log₂(2D/W)

T = 移动时间
D = 起点到目标的距离
W = 目标宽度

结论:
├── 距离越短，时间越短
└── 目标越大，时间越短
```

### 上下文工具

将操作工具放在内容中，使内容和操作融合。

#### 1. 实时可见工具

## 巧用过渡

人脑会对动态的事物保持敏感，适当的过渡效果能让界面生动，增强沟通。

### 动效价值

| 价值 | 说明 |
|------|------|
| **增加体验舒适度** | 让认知过程更自然 |
| **增加界面活力** | 吸引注意力，突出重点 |
| **描述层级关系** | 体现元素间的层级与空间关系 |
| **提供反馈** | 助力交互体验 |

### 动效原则

```
┌─────────────────────────────────────────────────────────┐
│                     动效三原则                            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   自然                                                  │
│   ├── 自然运动规律                                      │
│   ├── 视觉连贯性                                        │
│   └── 用户感知动作成立                                  │
│                                                         │
│   高效                                                  │
│   ├── 节省过渡时间                                      │
│   ├── 快速完成过渡                                      │
│   └── 出场比进场更快                                    │
│                                                         │
│   克制                                                  │
│   ├── 有意义的动效                                      │
│   ├── 不过度修饰                                        │
│   └── 不干扰用户                                        │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 过渡应用场景

#### 1. 保持上下文

| 过渡类型 | 场景 |
|---------|------|
| **滑入滑出** | 构建虚拟空间 |
| **传送带** | 扩展虚拟空间 |
| **折叠窗口** | 保持上下文，拓展空间 |

#### 2. 解释发生了什么

```
对象增加:
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   新增条目 → 高亮显示 → 几秒后恢复                       │
│                                                         │
│   ┌───┐                                                 │
│   │ 1 │                                                 │
│   ├───┤                                                 │
│   │ 2 │ ← 高亮                                          │
│   ├───┤                                                 │
│   │ 3 │                                                 │
│   └───┘                                                 │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

```
对象删除:
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   删除条目 → 淡出移除 → 其他条目上移                      │
│                                                         │
│   ┌───┐     ┌───┐     ┌───┐                            │
│   │ 1 │     │ 1 │     │ 1 │                            │
│   ├───┤     ├───┤     ├───┤                            │
│   │ 2 │ →   │   │ →   │ 3 │                            │
│   ├───┤     ├───┤     ├───┤                            │
│   │ 3 │     │ 3 │     │ 4 │                            │
│   └───┘     └───┘     └───┘                            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

```
对象更改:
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   更改内容 → 黄底高亮 → 几秒后恢复                       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

#### 3. 改善感知性能

当无法提升实际性能时，转移用户注意力来缩短感知时间。

## 提供邀请

很多富交互模式缺少易发现性，「提供邀请」是成功完成人机交互的关键。

### 核心概念

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   意符 (Signifiers):                                    │
│   额外提示，告诉用户可以采取什么行为                       │
│   必须可感知（视觉、听觉、触觉）                          │
│                                                         │
│   可供性 (Affordance):                                  │
│   物品特性与主体能力之间的关系                           │
│   部分可感知 (Perceived Affordance)                     │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 静态邀请

#### 1. 引导操作邀请

| 类型 | 示例 |
|------|------|
| **文本邀请** | "点击上传图片" |
| **白板式邀请** | 空白卡片 + "添加内容" |
| **未完成邀请** | 进度条 + "完善信息" |

#### 2. 漫游探索邀请

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   首次登录时出现少量「探索点」                            │
│                                                         │
│   ┌─────────────────────────────────────┐               │
│   │                    ┌───┐            │               │
│   │                    │ 1 │ ← 探索点   │               │
│   │                    └───┘            │               │
│   │  ┌───────────────────────────────┐  │               │
│   │  │  这是新功能说明               │  │               │
│   │  │                    [我知道了] │  │               │
│   │  └───────────────────────────────┘  │               │
│   └─────────────────────────────────────┘               │
│                                                         │
│   注意：保持简单，容易退出，内容简明                       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 动态邀请

#### 1. 悬停邀请

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   悬停时提供邀请                                         │
│                                                         │
│   普通状态:                    悬停状态:                  │
│   ┌───────────┐               ┌───────────┐             │
│   │ 卡片内容  │               │ 卡片内容  │             │
│   │           │       →       │           │             │
│   │           │               │ [查看详情]│ ← 按钮出现  │
│   └───────────┘               └───────────┘             │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

#### 2. 推论邀请

基于用户行为推断可能的需求：

```
用户点赞文章 → 系统推断可能喜欢同类 → 邀请开启相关推荐
```

#### 3. 更多内容邀请

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   在 Modal 中显示前后切换箭头                             │
│                                                         │
│   ┌─────────────────────────────────────┐               │
│   │ ◀                           ▶      │               │
│   │     当前内容                       │               │
│   │                                    │               │
│   └─────────────────────────────────────┘               │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 即时反应

用户进行操作后，系统应立即有对应的反馈。

### 牛顿第三定律

```
作用力 = 反作用力

用户操作 = 系统反馈

输入量级越大 → 反馈量级越大
重要性越高   → 反馈重要性越高
```

## 导航设计

导航告知用户在哪里、能去哪里、如何到达。

### 设计要点

1. 提供标识、上下文线索，避免迷路
2. 保持导航样式和行为一致
3. 尽可能减少页面跳转

### 导航类型

#### 顶部导航

```
┌─────────────────────────────────────────────────────────┐
│ [Logo]  首页  产品  解决方案  文档  定价        [登录]   │
└─────────────────────────────────────────────────────────┘

适用: 门户性质、前台应用
类目: 2-7 个
标题: 4-15 字符
```

#### 侧边导航

```
┌──────┬──────────────────────────────────────────────────┐
│      │                                                  │
│ 概览  │                                                  │
│      │                                                  │
│ 列表  │              内容区域                            │
│      │                                                  │
│ 分析  │                                                  │
│      │                                                  │
│ 设置  │                                                  │
│      │                                                  │
└──────┴──────────────────────────────────────────────────┘

适用: 管理性质、操作频率高
类目: 不限
```

#### 面包屑

```
首页 / 产品列表 / 产品详情

注意:
├── 层级不超过 5 级
├── 导航清晰时可不使用
└── 始终展示当前位置
```

#### 标签页

| 样式 | 使用场景 |
|------|---------|
| **基本样式** | 主功能切换 |
| **卡片样式** | 页面局部展示 |
| **胶囊样式** | 小版块内选项切换 |
| **竖状样式** | 分类较多的选项 |

#### 步骤条

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   ① ─────── ② ─────── ③ ─────── ④                    │
│  完成       进行中     等待       等待                   │
│                                                         │
│   横向: 2-5 步，每步 ≤12 字符                            │
│                                                         │
│   竖向: 较多步骤，可追加描述                              │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 反馈机制

当用户和系统需要交互时，使用不同模式反馈信息或结果。

### 反馈原则

1. 为用户提供必要、积极、即时的反馈
2. 避免过度反馈
3. 能直接看到效果的简单操作可省略反馈

### 反馈类型

#### 提示信息

| 组件 | 类型 | 特点 |
|------|------|------|
| **Alert** | 静态 | 非阻断，停留页面位置，不自动消失 |
| **Notification** | 浮动 | 右上角，重要全局通知 |
| **Badge** | 标记 | 图标右上角，聚合消息提示 |
| **Tooltip** | 悬停 | 简单描述，不承载复杂内容 |
| **Popover** | 悬停/点击 | 可承载链接、按钮等复杂内容 |

#### 过程反馈

| 组件 | 使用场景 |
|------|---------|
| **Spin** | 加载状态 |
| **Progress** | 操作进度 |
| **Popconfirm** | 轻量级确认 |

#### 结果反馈

| 组件 | 特点 | 使用场景 |
|------|------|---------|
| **Message** | 顶部居中，3s 自动消失 | 操作成功提示 |
| **Modal** | 页面居中，阻断操作 | 重要确认、失败通知 |

### 反馈选择指南

```
┌─────────────────────────────────────────────────────────┐
│                     反馈选择决策                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   操作成功？                                            │
│   ├── 是 → 操作重要？                                   │
│   │        ├── 是 → Message                             │
│   │        └── 否 → 可省略                               │
│   │                                                    │
│   └── 否 → 失败重要？                                   │
│            ├── 是 → Modal（说明原因）                   │
│            └── 否 → Message                             │
│                                                         │
│   需要确认？                                            │
│   ├── 简单确认 → Popconfirm                             │
│   └── 复杂确认 → Modal                                  │
│                                                         │
│   需要持久展示？                                        │
│   ├── 是 → Alert / Notification                        │
│   └── 否 → Message                                     │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 交互设计检查清单

### 足不出户

- [ ] 能在同一页面完成的操作不跳转
- [ ] 二次确认使用轻量级方式
- [ ] 详情查看使用覆盖层或嵌入层
- [ ] 复杂流程使用步骤条

### 简化交互

- [ ] 重要操作实时可见
- [ ] 次要操作悬停可见
- [ ] 热区足够大便于点击
- [ ] 移动端触控区域 ≥44px

### 巧用过渡

- [ ] 页面切换有过渡动画
- [ ] 状态变化有动效提示
- [ ] 动效自然、高效、克制
- [ ] 尊重用户减少动画设置

### 提供邀请

- [ ] 新功能有引导提示
- [ ] 操作入口清晰可发现
- [ ] 悬停时有视觉反馈
- [ ] 空状态有操作建议

### 即时反应

- [ ] 操作后有即时反馈
- [ ] 加载状态有指示
- [ ] 错误有清晰提示和恢复路径
- [ ] 成功有确认反馈

### 导航

- [ ] 用户知道当前位置
- [ ] 导航层级不超过 5 级
- [ ] 返回行为可预测
- [ ] 导航样式一致

### 反馈

- [ ] 反馈及时且恰当
- [ ] 不打断用户流程
- [ ] 错误信息明确
- [ ] 提供解决方案
