Ant Design 提供了一套完整的页面模板，帮助企业级应用快速搭建界面。

## 表单页

表单页用于信息添加、录入，确保用户按要求提交信息。

### 设计目标

- 明确当前页面任务
- 快速查找和定位修改目标
- 轻松准确地理解表单项含义
- 简化填写流程

### 设计原则

```
┌─────────────────────────────────────────────────────────┐
│                    表单页设计原则                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   高效                                                  │
│   ├── 合理的信息组织形式                                │
│   └── 正确的表单组件使用                                │
│                                                         │
│   明确                                                  │
│   ├── 快速定位重要信息                                  │
│   ├── 内容准确传达含义                                  │
│   └── 及时响应相关反馈                                  │
│                                                         │
│   安全感                                                │
│   ├── 分步保存机制                                      │
│   ├── 返回、重置、撤销功能                              │
│   └── 避免数据丢失                                      │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 模板类型

#### 1. 基础表单

```
┌─────────────────────────────────────────────────────────┐
│ 新建项目                                                 │

│ │                                                     │ │

│ │                                                     │ │
│ └─────────────────────────────────────────────────────┘ │
│                                                         │
│ 项目类型                                                │
│ ┌───────────────┐                                      │
│ │ 请选择    ▼  │                                      │
│ └───────────────┘                                      │
│                                                         │
│                          [取消]  [提交]                │
```

**使用场景**：
- 内容项较少
- 内容项无法按相关性分组
- 简单快速的任务

```

```

## 详情页

详情页向用户展示一个对象的完整信息，主要用于信息浏览。

### 设计目标

提高信息浏览和搜寻效率，便捷执行操作。

### 设计原则

```
┌─────────────────────────────────────────────────────────┐
│                    详情页设计原则                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   直截了当                                              │
│   ├── 信息尽量平铺展示                                  │
│   └── 无必要不做大量隐藏、折叠                          │
│                                                         │
│   层次分明                                              │
│   ├── 信息分层分组展示                                  │
│   └── 降低单个页面信息复杂度                            │
│                                                         │
│   化繁为简                                              │
│   ├── 减少复杂结构使用                                  │
│   ├── 使用相似结构和模块                                │
│   └── 让用户更聚焦信息本身                              │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 模板类型

#### 1. 基础详情


## 数据列表

数据列表用于展示和管理大量数据。

### 设计目标

- 易于扫读
- 快速查找对象

### 列表类型

| 类型 | 特点 | 使用场景 |
|------|------|---------|
| **表格** | 矩阵布局，对比分析 | 复杂数据，多列展示 |
| **列表** | 垂直排列，快速扫读 | 有限空间，移动友好 |
| **卡片列表** | 网格布局，视觉强调 | 视觉内容，平等展示 |

```
表格:
┌─────────┬─────────┬─────────┬─────────┐
│  列1    │  列2    │  列3    │  操作   │
├─────────┼─────────┼─────────┼─────────┤
│  数据   │  数据   │  数据   │ 编辑 删除│
├─────────┼─────────┼─────────┼─────────┤
│  数据   │  数据   │  数据   │ 编辑 删除│
└─────────┴─────────┴─────────┴─────────┘

列表:
┌─────────────────────────────────────┐
│ [头像] 标题                          │
│        描述文字                      │
├─────────────────────────────────────┤
│ [头像] 标题                          │
│        描述文字                      │
└─────────────────────────────────────┘

卡片列表:
┌───────────┐ ┌───────────┐ ┌───────────┐
│           │ │           │ │           │
│   卡片    │ │   卡片    │ │   卡片    │
│           │ │           │ │           │
└───────────┘ └───────────┘ └───────────┘
```

### 列表工具栏

```
┌─────────────────────────────────────────────────────────┐
│ [🔍 搜索...]    [筛选 ▼]    [排序 ▼]        [+ 新建]   │
├─────────────────────────────────────────────────────────┤
│                                                         │
│                      列表内容                           │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**工具栏功能**：
- 搜索
- 筛选
- 排序
- 视图切换
- 批量操作
- 新建

### 操作行为

#### 搜寻数据

| 方式 | 特点 |
|------|------|
| **查询** | 选择条件后一次性提交 |
| **筛选** | 调整筛选项结果即变化 |
| **搜索** | 输入关键词一次查询多种属性 |

#### 导航至详情

| 方式 | 使用场景 |
|------|---------|
| **同页展开** | 信息不超过一屏 |
| **抽屉展开** | 信息超过一屏 |
| **跳转独立页** | 需要分享，沉浸式任务 |
| **双栏展示** | 快速切换查看 |

#### 批量操作

```
┌─────────────────────────────────────────────────────────┐
│ ☑ 已选择 3 项                          [取消选择]      │
├─────────────────────────────────────────────────────────┤
│ [批量删除]  [批量导出]  [批量分配]                      │
└─────────────────────────────────────────────────────────┘
```

#### 新建

| 方式 | 使用场景 |
|------|---------|
| **右上角新建按钮** | 通用入口，创建后显示在列表第一条 |
| **虚线新建按钮** | 位置固定，创建后显示在该位置 |

#### 删除

| 方式 | 使用场景 |
|------|---------|
| **直接删除** | 允许撤销 |
| **二次确认** | 需要确认 |
| **安全校验** | 高安全级别操作 |

### 布局结构

```
┌─────────────────────────────────────────────────────────┐
│ 页面标题                                      [+ 新建]  │
├─────────────────────────────────────────────────────────┤
│ 独占式区域（可选）：复杂搜寻、数据统计                   │
├─────────────────────────────────────────────────────────┤
│ [搜索...]  [筛选]  [排序]  [视图]                       │
├─────────────────────────────────────────────────────────┤
│ ☐ 列表项 1                                   [操作]    │
│ ☐ 列表项 2                                   [操作]    │
│ ☐ 列表项 3                                   [操作]    │
├─────────────────────────────────────────────────────────┤
│ 共 100 条  < 1 2 3 4 5 >                               │
└─────────────────────────────────────────────────────────┘
```

## 空状态

任何内容区域没有内容/数据时，展示空状态。

### 设计目标

- 告知空状态原因
- 帮助用户摆脱空状态

### 设计原则

```
┌─────────────────────────────────────────────────────────┐
│                    空状态设计原则                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   明确                                                  │
│   ├── 清晰明了的语言                                    │
│   └── 插画等形式告知具体原因                            │
│                                                         │
│   提供邀请                                              │
│   ├── 帮助文案                                          │
│   ├── 建议操作                                          │
│   └── 引导用户进行操作                                  │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 类型

#### 1. 新手引导

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│                       [插画]                            │
│                                                         │
│                 欢迎使用项目管理                         │
│                                                         │
│         创建您的第一个项目，开始高效协作                 │
│                                                         │
│                    [+ 创建项目]                         │
│                                                         │
│                                                         │
│    ○───────────────────────────────────────────○       │
│    1                                             2      │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**使用场景**：首次使用应用或功能

**组成**：
- 状态提示
- 帮助引导
- 建议操作

#### 2. 完成/清空

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│                       [插画]                            │
│                                                         │
│                   太棒了，全部完成！                     │
│                                                         │
│              您已处理完所有待办事项                      │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**使用场景**：
- 任务清单全部完成
- 全部通知已读

**特点**：图形元素 + 提示信息，无需操作引导

#### 3. 无数据

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│                       [插画]                            │
│                                                         │
│                   暂无数据                              │
│                                                         │
│            没有找到符合条件的搜索结果                    │
│                                                         │
│                    [清除筛选]                           │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**使用场景**：内容区域无数据

**组成**：
- 图形元素
- 提示信息
- 建议操作（可选）

## 异常页

用于展示页面异常状态。

### 设计目标

解释异常，提供相应建议或操作，避免用户迷失和困惑。

### 设计原则

```
┌─────────────────────────────────────────────────────────┐
│                    异常页设计原则                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   友好                                                  │
│   ├── 友好、清晰的语言                                  │
│   └── 避免难懂的术语                                    │
│                                                         │
│   提供邀请                                              │
│   ├── 引导下一步交互                                   │
│   └── 表明可以做什么                                    │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 类型

#### 404 页面

```

│                       [插画]                            │
│                                                         │
```

### 异常页要素

1. **配图**：增添乐趣，缓解烦躁
2. **异常代码**：有具体 HTTP 错误代码时展示
3. **异常描述**：简明扼要描述原因
4. **建议操作**：引导用户处理或回到正确路径
