## 布局系统

空间布局是体系化视觉设计的起点，基于「秩序之美」的原则。

### 核心要素

| 要素 | 说明 |
|------|------|
| 统一画板 | 1440px |
| 适配方案 | 左右布局 / 上下布局 |
| 网格单位 | 8px 基数 |
| 栅格系统 | 24 栅格 |
| 常用模度 | 8 倍数双数组 |

### 统一画板

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   设计稿画板尺寸: 1440px                                │
│                                                         │
│   主流分辨率: 1920 / 1440 / 1366 / 1280                 │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 适配方案

#### 左右布局适配

```
┌──────┬──────────────────────────────────────────────────┐
│      │                                                  │
│      │                                                  │
│ 固定  │              动态缩放                            │
│ 导航  │              内容区域                            │
│      │                                                  │
│      │                                                  │
└──────┴──────────────────────────────────────────────────┘
```

**适用场景**：管理后台、侧边导航应用

#### 上下布局适配

```
┌──────────────────────────────────────────────────────────┐
│                      顶部导航                             │
├──────────────────────────────────────────────────────────┤
│  │                                                    │  │
│  │                    内容区域                         │  │
│  │                                                    │  │
│  留白                                                 留白
└──────────────────────────────────────────────────────────┘
```

**适用场景**：门户首页、展示型页面

### 网格单位

**基数：8px**

```
网格体系: 8px 为最小单位

优势:
├── 符合偶数思维
├── 匹配主流显示设备
├── 快速决策布局
└── 简化设计到开发的沟通
```

### 栅格系统

**24 栅格体系**

```
┌──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬──┐
│  │  │  │  │  │  │  │  │  │  │  │  │  │  │  │  │  │  │  │  │  │  │  │  │
│  │  │  │  │  │  │  │  │  │  │  │  │  │  │  │  │  │  │  │  │  │  │  │  │
└──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──┘

Column: 宽度随浏览器变化
Gutter: 固定值
```

**设计者与开发者沟通要点**：

1. 清晰定义动态布局范围
2. 尽量保持偶数思维
3. 关键数据交付（Gutter、Column）
4. 区块从 column 开始到 column 结束

### 常用模度

**8 倍数双数组**

```
推荐间距值:
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   小号间距: 8px                                          │
│   中号间距: 16px                                         │
│   大号间距: 24px                                         │
│                                                         │
│   间距公式: y = 8 + 8 × n (n ≥ 0)                        │
│                                                         │
│   常用组合: 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48 / 64    │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 色彩系统

Ant Design 将色彩体系解读为两个层面：系统级色彩体系和产品级色彩体系。

### 色彩模型

**HSB 色彩模型**：便于设计师调整颜色时有明确心理预期。

- H (Hue): 色相
- S (Saturation): 饱和度
- B (Brightness): 明度

### 系统级色彩体系

#### 基础色板

**120 个颜色 = 12 主色 + 衍生色**

| 主色 | 用途 |
|------|------|
| Blue | 品牌色、信息 |
| Red | 错误、危险 |
| Green | 成功 |
| Gold | 警告 |
| Cyan | 辅助信息 |
| Purple | 特殊标记 |
| ... | ... |

#### 中性色板

**13 个从白到黑的颜色**

```
白色 ────────────────────────────────────────────────── 黑色

#FFFFFF → ... → #000000

用途:
├── 文字 (主文字、次文字、辅助文字、禁用文字)
├── 边框
├── 分割线
└── 背景
```

#### 功能色

| 功能 | 颜色 | 使用场景 |
|------|------|---------|
| **成功** | Green | 操作成功、完成状态 |
| **错误** | Red | 错误提示、危险操作 |
| **警告** | Gold | 警告提示、注意事项 |
| **信息** | Blue | 信息提示、链接 |

### 产品级色彩体系

#### 品牌色应用

```
品牌色选择规则:
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   从基础色板选择主色                                     │
│   └── 建议选择从浅至深的第 6 个颜色                      │
│                                                         │
│   Ant Design 品牌色: #1677ff                            │
│                                                         │
│   应用场景:                                              │
│   ├── 关键行动点                                         │
│   ├── 操作状态                                           │
│   ├── 重要信息高亮                                       │
│   └── 图形化元素                                         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

#### 中性色应用

| 类型 | 透明度 | 用途 |
|------|--------|------|
| 主文字 | 85% | 标题、正文 |
| 次文字 | 65% | 次要信息 |
| 辅助文字 | 45% | 说明文字 |
| 禁用文字 | 25% | 禁用状态 |
| 边框 | 15% | 边框线 |
| 分割线 | 6% | 分割线 |
| 背景 | 4% | 布局背景 |

### 色彩使用原则

1. **克制**：色彩用于信息传递、操作引导、交互反馈
2. **一致**：功能色保持统一，不要自定义干扰认知
3. **可访问**：符合 WCAG 2.0 标准

## 字体系统

科学的字体系统大大提升用户的阅读体验及工作效率。

### 字体家族

```css
font-family: 
  -apple-system,           /* iOS/macOS */
  BlinkMacSystemFont,      /* macOS */
  'Segoe UI',              /* Windows */
  Roboto,                  /* Android */
  'Helvetica Neue',        /* macOS */
  Arial,                   /* 通用 */
  'Noto Sans',             /* 多语言支持 */
  sans-serif,
  'Apple Color Emoji',     /* Emoji */
  'Segoe UI Emoji',
  'Segoe UI Symbol',
  'Noto Color Emoji';
```

### 主字体

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   基于:                                                  │
│   ├── 显示器阅读距离: 50cm                               │
│   └── 最佳阅读角度: 0.3                                  │
│                                                         │
│   主字体: 14px                                           │
│   行高: 22px                                             │
│                                                         │
│   升级原因: 保证多数常用显示器上的阅读效率最佳             │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 字阶与行高

**10 个字阶 + 对应行高**（受 5 音阶及自然律启发）

| 字阶 | 行高 | 用途 |
|------|------|------|
| 12px | 20px | 辅助文字、注释 |
| 14px | 22px | 正文、标签 |
| 16px | 24px | 小标题 |
| 20px | 28px | 标题 |
| 24px | 32px | 大标题 |
| 30px | 38px | 特大标题 |
| 38px | 46px | 展示型标题 |
| 46px | 54px | 展示型大标题 |
| 56px | 64px | 展示型特大标题 |
| 68px | 76px | 数据展示 |

**字阶选择建议**：
- 展示型页面除外
- 字阶选择控制在 3-5 种
- 保持克制的原则

### 字重

| 字重 | 数值 | 用途 |
|------|------|------|
| Regular | 400 | 正文 |
| Medium | 500 | 标签、强调 |
| Semibold | 600 | 英文加粗 |

### 字体颜色

**参考 WCAG 标准，保持 AAA 级对比度（7:1 以上）**

| 类型 | 用途 |
|------|------|
| 主文字 | 标题、重要信息 |
| 次文字 | 次要信息、描述 |
| 辅助文字 | 说明、提示 |
| 禁用文字 | 禁用状态 |

### 字体建议

1. **体系化**：统一规划主、次、辅助、标题、展示等类别
2. **少即是多**：尽量少的样式实现设计目的
3. **韵律感**：字阶跳跃选择产生微妙韵律

## 阴影系统

阴影来源于现实生活的反映物体与物体之间距离的物理现象。

### 高度层级

```
┌─────────────────────────────────────────────────────────┐
│                    UI 高度层级                           │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   第 3 层: 对话框、Modal                     ┌───────┐  │
│                                             │       │  │
│   第 2 层: 下拉面板、Popover           ┌────┴───┐   │  │
│                                       │        │   │  │
│   第 1 层: 悬浮卡片 (hover)      ┌────┴────────┴───┐│  │
│                                 │                  ││  │
│   第 0 层: 输入框、按钮   ─────────────────────────────│
│                                                         │
└─────────────────────────────────────────────────────────┘
```

| 层级 | 阴影 | 示例组件 |
|------|------|---------|
| **第 0 层** | 无阴影 | 输入框、按钮 |
| **第 1 层** | 轻微阴影 | 卡片 hover |
| **第 2 层** | 中等阴影 | Dropdown、Popover |
| **第 3 层** | 较重阴影 | Modal、Drawer |

### 阴影方向

| 方向 | 使用场景 |
|------|---------|
| **向下** | 组件内部或组件本身（常规） |
| **向上** | 底部导航、工具栏 |
| **向左** | 右边导航栏、抽屉、固定表格栏 |
| **向右** | 左边导航栏、抽屉、固定表格栏 |

### 三层阴影叠加

**Ant Design 4.0 采用三层阴影，更柔和真实**

```
阴影值组成:
├── 第一层: 深色、小模糊
├── 第二层: 中色、中模糊
└── 第三层: 浅色、大模糊

效果: 越远的物体，阴影越淡、模糊越高、面积越大
```

### 阴影设计表

**第 1 层示例**:
```css
/* 向下 */
box-shadow: 
  0 1px 2px -2px rgba(0, 0, 0, 0.16),
  0 3px 6px 0 rgba(0, 0, 0, 0.12),
  0 5px 12px 4px rgba(0, 0, 0, 0.09);
```

**第 2 层示例**:
```css
/* 向下 */
box-shadow: 
  0 3px 6px -4px rgba(0, 0, 0, 0.12),
  0 6px 16px 0 rgba(0, 0, 0, 0.08),
  0 9px 28px 8px rgba(0, 0, 0, 0.05);
```

**第 3 层示例**:
```css
/* 向下 */
box-shadow: 
  0 6px 16px -8px rgba(0, 0, 0, 0.08),
  0 9px 28px 0 rgba(0, 0, 0, 0.05),
  0 12px 48px 16px rgba(0, 0, 0, 0.03);
```

## 图标系统

图标是 UI 设计中必不可少的组成，将概念转换成清晰易读的图形。

### 设计原则

| 原则 | 说明 |
|------|------|
| **准确** | 造型准确（偶数原则），表意准确 |
| **简单** | 表意清晰基础上，尽量简洁 |
| **节奏** | 挖掘构图中的秩序之美 |
| **愉悦** | 赋予适度情感 |

### 设计规格

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   画板: 1024 × 1024px                                   │
│                                                         │
│   ┌────────────────────────────────────────┐            │
│   │  ←────── 64px 出血位 ──────→           │            │
│   │  ┌──────────────────────────────┐      │            │
│   │  │                              │      │            │
│   │  │        图标内容区域           │      │            │
│   │  │        (安全区域)             │      │            │
│   │  │                              │      │            │
│   │  └──────────────────────────────┘      │            │
│   │                                        │            │
│   └────────────────────────────────────────┘            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 元素规格

#### 点

| 规格 | 用途 |
|------|------|
| 80px | 小点 |
| 96px | 常用 |
| 112px | 大点 |
| 128px | 特大 |

#### 线

| 规格 | 用途 |
|------|------|
| 56px | 细线 |
| 64px | 常用 |
| 72px | 中线 |
| 80px | 粗线 |

#### 圆角

| 规格 | 用途 |
|------|------|
| 8px | 小圆角 |
| 16px | 常用 |
| 32px | 大圆角 |

**注意**：图标内部空间保持直角

#### 角度

**常用角度：约 76°**（受美式战斗机 F-14 Tomcat 启发）

### 秩序之美

#### 1. 形式

```
基础图形演变:
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   ○ 圆 ─────────────→ 圆形图标                          │
│   □ 方 ─────────────→ 方形图标                          │
│   △ 三角 ───────────→ 三角形图标                        │
│                                                         │
│   理性构图，而非直觉式设计                               │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

#### 2. 韵律

**元素韵律**：保持同一元素的规格规律

**构图韵律**：类似图标构造一致，比例关系理性

#### 3. 平衡

```
视觉平衡调整:
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   弯曲线条比竖直线条看起来细                              │
│   └── 圆形边框微调 +4px                                  │
│                                                         │
│   倾斜线条比竖直线条看起来细                              │
│   └── 倾斜线条微调 +4px                                  │
│                                                         │
│   留白空间影响视觉重量                                    │
│   └── 通过调节线条粗细达到平衡                            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

#### 4. 辨识

**让科技有温度**：
- 圆角 32px（而非过圆的 72）
- 看起来坚硬理性，但不过于尖锐

**让图形有生命**：
- 适度注入拟人化元素
- 图标具备生命力

### 图标设计建议

1. 干掉多余节点，保持图形整洁
2. 合并图形，便于输出
3. 走查小数点和奇数
4. 整洁的图层管理

## 暗黑模式

暗黑模式是指把所有 UI 换成黑色或者深色的一个主题模式。

### 使用场景

1. **暗光环境工作**：减小亮度对比带来的视觉压力
2. **沉浸式专注**：帮助使用者更关注内容

### 设计原则

| 原则 | 说明 |
|------|------|
| **内容舒适性** | 避免对比很强的色彩，长时间使用不疲劳 |
| **信息一致性** | 与浅色模式保持层级关系一致 |

### 色彩处理

基于基础色板 + 透明度规则，使色彩在不同环境色下更好融合。

```
暗黑模式色彩策略:
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   1. 使用透明度而非反转颜色                               │
│                                                         │
│   2. 降低饱和度，避免过亮                                 │
│                                                         │
│   3. 保持层级关系：                                      │
│      背景 - 内容 - 浮层 - 弹窗                           │
│      (由深到浅)                                         │
│                                                         │
│   4. 文字对比度仍需达标 (≥4.5:1)                         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 暗黑模式检查

- [ ] 背景色深浅层次正确
- [ ] 文字对比度达标
- [ ] 功能色（成功/错误/警告）可辨识
- [ ] 禁用状态清晰
- [ ] 边框和分割线可见
- [ ] 阴影效果适当调整
