## 1. 自然 (Natural)

数字世界的光速迭代使得产品日益复杂，而人类意识和注意力资源有限。追求「自然」交互是 Ant Design 持之以恒的方向。

### 感知自然

约 80% 外界信息通过视觉通道获取。界面设计应充分汲取自然界规律。

**视觉元素自然化**：

| 元素 | 自然规律应用 |
|------|-------------|
| **布局** | 符合阅读习惯（Z型、F型扫描） |
| **色彩** | 源于自然场景，渐变而非跳跃 |
| **插画** | 真实感、熟悉感 |
| **图标** | 抽象自真实物体 |
| **动效** | 遵循物理运动规律 |

**多感官设计**：
- 听觉：提示音、反馈音
- 触觉：震动反馈（移动端）

### 行为自然

设计应理解用户、系统角色、任务目标间的关系。

**主动式服务策略**：
- 行为分析
- 人工智能辅助
- 传感器数据
- 元数据利用

**目标**：帮助用户决策、减少操作，节约脑力和体力。

## 2. 确定性 (Certainty)

界面是用户与系统交互的媒介。Ant Design 创造的产品界面应是高确定性、低合作熵的状态。

### 设计者确定

企业级产品是分工合作的产物，参与者越多合作熵越高。

**三大方法**：

#### 1. 保持克制

> "完美不在于无以复加，而在于无可删减。" — Antoine de Saint-Exupéry

- 聚焦最有价值的功能
- 用最少的设计元素表达
- 能做，但想清楚不做

#### 2. 面向对象的方法

探索设计规律，抽象成「对象」：

| 领域 | 抽象对象 |
|------|---------|
| 颜色 | 色板生成算法 |
| 间距 | 栅格系统 |
| 字体 | 字阶系统 |
| 组件 | 组件 API |

**优势**：增强灵活性和可维护性，减少主观干扰。

#### 3. 模块化设计

将复杂或重复的局部封装成模块：

- 提供有限接口
- 降低系统复杂度
- 增进可靠性和可维护性
- 把创造力专注在最需要的地方

### 用户确定

用户通过多个企业级产品协同完成工作，需要跨产品一致性。

**一致性维度**：

| 维度 | 内容 |
|------|------|
| **外观一致** | 视觉风格、色彩、图标 |
| **交互一致** | 操作方式、反馈模式 |
| **文案一致** | 术语、语气、格式 |
| **结构一致** | 导航、布局、信息架构 |

**价值**：提升易学性，降低认知和操作成本。

### 合作熵控制

```
┌─────────────────────────────────────────────────────────┐
│                   合作熵来源与对策                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   来源                    对策                           │
│   ────                    ────                           │
│   设计风格不统一    →    设计规范 + 组件库               │
│   沟通成本高        →    模块化 + 清晰接口               │
│   重复设计          →    可复用组件/模板                 │
│   主观差异          →    数据驱动决策                    │
│   文档缺失          →    设计文档 + 使用指南             │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 3. 意义感 (Meaningfulness)

产品设计应充分站在工作视角，促成用户使命的达成，同时兼顾人性需求。

### 结果的意义：明确目标，即时反馈

**设计要点**：

1. **目标清晰**
   - 洞悉工作目标
   - 拆解明确的子目标
   - 每个交互围绕主目标

2. **反馈即时**
   - 每个行为有恰当反馈
   - 用户对结果了然于胸

3. **情感化设计**
   - 安抚负面情感
   - 强化正面情感

### 过程的意义：挑战适中，全情投入

**心流状态设计**：

```
┌─────────────────────────────────────────────────────────┐
│                     心流通道                              │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   挑战 │                    心流区                      │
│   程度 │                      ▓▓▓                       │
│        │                    ▓▓▓▓▓▓▓                     │
│   高   │                  ▓▓▓▓▓▓▓▓▓▓▓                   │
│        │              焦虑│▓▓▓▓▓▓▓▓▓▓▓│无聊            │
│        │                  │▓▓▓▓▓▓▓▓▓▓▓│                │
│   低   │                  └───────────┘                │
│        └─────────────────────────────────── 技能水平   │
│              低                          高             │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**设计原则**：

- 功能适时适地触发
- 匹配用户能力
- 如无必要，勿增实体
- 让用户专注任务而非界面

### 意义感检查

| 检查项 | 问题 |
|--------|------|
| **目标** | 用户知道要完成什么吗？ |
| **反馈** | 用户知道发生了什么吗？ |
| **挑战** | 任务难度适中吗？ |
| **干扰** | 有不必要的干扰吗？ |
| **成就** | 完成后有成就感吗？ |

## 4. 生长性 (Growth)

企业级产品功能的增长与用户系统角色的演变相生相伴。

### 价值连接

产品的增长依赖于用户的群体扩大和深度使用，用户的成长又依赖于产品功能的完善。

**连接策略**：

| 策略 | 实践 |
|------|------|
| **价值发现** | 让隐藏功能被发现 |
| **场景挖掘** | 探索不同场景下的需求 |
| **渐进引导** | 从简单到复杂的使用路径 |
| **价值证明** | 展示产品带来的收益 |

### 人机共生

将用户和系统作为一个动态发展的共同体来思考。

**设计考虑**：

```
┌─────────────────────────────────────────────────────────┐
│                   人机共生设计                            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   用户侧                    系统侧                       │
│   ──────                    ──────                       │
│   新手用户              →   简化功能、引导教学           │
│   熟练用户              →   效率工具、快捷操作           │
│   专家用户              →   高级功能、自定义能力         │
│                                                         │
│   功能侧                                                 │
│   ──────                                                 │
│   基础功能              →   稳定、可靠                   │
│   进阶功能              →   渐进披露                     │
│   高级功能              →   按需发现                     │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**共生特征**：
- 足够灵活
- 足够包容
- 充满生命力

### 功能可发现性

**设计方法**：

1. **渐进式展现**
   - 常用功能明显
   - 高级功能可发现
   - 隐藏功能有入口

2. **上下文引导**
   - 在使用场景中提示
   - 根据行为推荐
   - 智能预测需求

3. **成长路径**
   - 新手引导
   - 能力解锁
   - 成就系统

## 价值观应用决策

### 决策框架

```
┌─────────────────────────────────────────────────────────┐
│                 设计决策价值观框架                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   面临设计选择时，按以下顺序考量：                        │
│                                                         │
│   1. 自然性：这是用户期望的方式吗？                       │
│      └── YES → 继续                                     │
│      └── NO  → 寻找更自然的方案                          │
│                                                         │
│   2. 确定性：用户能预测结果吗？                          │
│      └── YES → 继续                                     │
│      └── NO  → 增加一致性、可预测性                      │
│                                                         │
│   3. 意义感：这有意义吗？                                │
│      └── YES → 继续                                     │
│      └── NO  → 明确目标、优化反馈                        │
│                                                         │
│   4. 生长性：能适应用户成长吗？                          │
│      └── YES → 实施                                     │
│      └── NO  → 增加灵活性、可扩展性                      │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 典型场景应用

| 场景 | 自然 | 确定性 | 意义感 | 生长性 |
|------|------|--------|--------|--------|
| **新功能设计** | 符合用户习惯 | 保持一致性 | 解决真实问题 | 渐进引导 |
| **交互优化** | 降低认知负担 | 可预测行为 | 即时反馈 | 适应不同水平 |
| **界面改版** | 保持熟悉感 | 继承交互模式 | 保留核心价值 | 兼容旧数据 |
| **错误处理** | 友好提示 | 明确原因 | 提供解决方案 | 预防机制 |

---

## 总结

| 价值观 | 核心问题 | 设计目标 |
|--------|----------|----------|
| **自然** | 用户觉得自然吗？ | 降低认知成本，直觉操作 |
| **确定性** | 用户能预测吗？ | 一致、可预期、可学习 |
| **意义感** | 用户有成就感吗？ | 目标明确，反馈即时，心流体验 |
| **生长性** | 能一起成长吗？ | 价值发现，灵活包容，持续进化 |

四大价值观相互支撑、互为补充，共同构成 Ant Design 设计体系的核心。
